# ユースケース: ダッシュボード監視

## 起動コマンド

```bash
bunx oh-my-ag dashboard
bunx oh-my-ag dashboard:web
```

Web ダッシュボードのデフォルト URL: `http://localhost:9847`

## 推奨ターミナルレイアウト

少なくとも3つのターミナルを使用してください:

1. ターミナルダッシュボード（`oh-my-ag dashboard`）
2. エージェントスポーンコマンド
3. テスト/ビルドログ

チームセッション時の共有可視性のために Web ダッシュボードを開いたままにしてください。

## ダッシュボードの監視対象

データソース: `.serena/memories/`

主要なシグナル:

- セッションステータス（`running`、`completed`、`failed`）
- タスクボードの割り当てと状態変更
- エージェントごとの進捗ターン
- 結果公開イベント

更新はファイル変更によるイベント駆動です。ディレクトリ全体のポーリングループは不要です。

## ランブック: シグナル → アクション

- `No agents detected`
  - エージェントが同じ `session-id` でスポーンされたか確認
  - `.serena/memories/` に書き込みが行われているか確認
- `Session stuck in running`
  - 最新の `progress-*` ファイルのタイムスタンプを確認
  - より明確なプロンプトで失敗またはブロックされたエージェントを再起動
- `Frequent reconnects (web)`
  - ローカルファイアウォール/プロキシを確認
  - `dashboard:web` を再起動してページを再読み込み
- `Missing activity while agents are active`
  - オーケストレーターの書き込みが別のワークスペースにリダイレクトされていないか確認

## マージ前の監視チェックリスト

- 必要なすべてのエージェントが完了状態に到達
- 未解決の高重大度 QA 指摘がない
- 各エージェントの最新の結果ファイルが存在
- 最終エージェント出力後に統合テストが実行済み

## 完了基準

監視フェーズは以下の条件を満たした場合に完了です:

- セッションが終了状態に到達（`completed` または意図的に停止）
- アクティビティ履歴が最終出力の来歴を説明できる
- 完全なステータス可視性をもってリリース/マージの判断が行われた
