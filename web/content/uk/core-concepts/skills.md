# Навички

## Поступове розкриття

Навички обираються на основі наміру запиту. Ручний вибір навичок зазвичай не потрібен.

## Дворівнева архітектура

Кожна навичка використовує **дворівневу архітектуру, оптимізовану за токенами**:

| Рівень | Вміст | Розмір |
|--------|-------|--------|
| `SKILL.md` | Ідентичність, умови маршрутизації, основні правила | ~40 рядків (~800B) |
| `resources/` | Протоколи виконання, приклади, чек-листи, плейбуки, фрагменти коду, технічний стек | Завантажується за потребою |

Це забезпечує **~75% економії токенів** при початковому завантаженні навички (3-7KB -> ~800B на навичку).

## Спільний рівень ресурсів (`_shared/`)

Загальні ресурси, дедупліковані між усіма навичками:

| Ресурс | Призначення |
|--------|-------------|
| `reasoning-templates.md` | Структуровані шаблони для багатокрокового міркування |
| `clarification-protocol.md` | Коли запитувати, а коли припускати; рівні неоднозначності |
| `context-budget.md` | Стратегії читання файлів з економією токенів для кожного рівня моделі |
| `context-loading.md` | Відповідність типу задачі до ресурсу для побудови промпту оркестратора |
| `skill-routing.md` | Відповідність ключових слів до навичок та правила паралельного виконання |
| `difficulty-guide.md` | Оцінка Просте/Середнє/Складне з розгалуженням протоколу |
| `lessons-learned.md` | Накопичені між сесіями доменні нюанси |
| `verify.sh` | Автоматизований скрипт верифікації, що запускається після завершення роботи агента |
| `api-contracts/` | PM створює контракти, backend реалізує, frontend/mobile використовують |
| `serena-memory-protocol.md` | Протокол читання/запису пам'яті в режимі CLI |
| `common-checklist.md` | Універсальні перевірки якості коду |

## Ресурси окремих навичок

Кожна навичка надає доменно-специфічні ресурси:

| Ресурс | Призначення |
|--------|-------------|
| `execution-protocol.md` | 4-кроковий робочий процес ланцюга міркувань (Аналіз -> План -> Реалізація -> Верифікація) |
| `examples.md` | 2-3 приклади вхідних/вихідних даних |
| `checklist.md` | Доменно-специфічний чек-лист самоперевірки |
| `error-playbook.md` | Відновлення після збоїв із правилом ескалації "3 спроби" |
| `tech-stack.md` | Детальні специфікації технологій |
| `snippets.md` | Готові до копіювання шаблони коду |

## Чому це важливо

Це дозволяє тримати початковий контекст мінімальним, водночас підтримуючи глибоке виконання за потреби.
