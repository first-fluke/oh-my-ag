# Робочі процеси

## Команди робочих процесів

- `/coordinate`
- `/orchestrate`
- `/plan`
- `/review`
- `/debug`

## Навички vs Робочі процеси

- Навички: автоматично активуються на основі наміру запиту
- Робочі процеси: явні багатокрокові конвеєри, що запускаються користувачем

## Типова послідовність мульти-агентної роботи

1. `/plan` для декомпозиції
2. `/coordinate` для поетапного виконання
3. `agent:spawn` для паралельних під-агентів
4. `/review` для контрольної точки QA
