# Сценарій використання: Моніторинг панелі

## Команди запуску

```bash
bunx oh-my-ag dashboard
bunx oh-my-ag dashboard:web
```

URL веб-панелі за замовчуванням: `http://localhost:9847`

## Рекомендоване розташування терміналів

Використовуйте щонайменше 3 термінали:

1. термінальна панель моніторингу (`oh-my-ag dashboard`)
2. команди запуску агентів
3. логи тестування/збірки

Тримайте веб-панель відкритою для спільної видимості під час командних сесій.

## Що відстежують панелі моніторингу

Джерело даних: `.serena/memories/`

Основні сигнали:

- статус сесії (`running`, `completed`, `failed`)
- дошка задач: призначення та зміни стану
- ходи прогресу кожного агента
- події публікації результатів

Оновлення базуються на подіях зміни файлів; повний цикл опитування директорії не потрібен.

## Ранбук: сигнал -> дія

- `No agents detected`
  - перевірте, що агенти були запущені з тим самим `session-id`
  - переконайтеся, що `.serena/memories/` записується
- `Session stuck in running`
  - перевірте мітки часу останнього файлу `progress-*`
  - перезапустіть збійного або заблокованого агента з чіткішим промптом
- `Frequent reconnects (web)`
  - перевірте локальний фаєрвол/проксі
  - перезапустіть `dashboard:web` та повторно відкрийте сторінку
- `Missing activity while agents are active`
  - перевірте, що записи оркестратора не перенаправлені в інший робочий простір

## Чек-лист моніторингу перед злиттям

- усі необхідні агенти досягли стану completed
- немає невирішених критичних знахідок QA
- наявні останні файли результатів для кожного агента
- інтеграційні тести виконані після фінальних результатів агентів

## Критерії завершення

Фаза моніторингу завершена, коли:

- сесія досягла термінального стану (`completed` або навмисно зупинена)
- історія активності пояснює походження фінального результату
- рішення про реліз/злиття прийнято з повною видимістю статусу
