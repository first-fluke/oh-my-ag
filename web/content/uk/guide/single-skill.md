# Сценарій використання: Одна навичка

## Коли використовувати цей шлях

Використовуйте цей варіант, коли результат має вузький обсяг і переважно належить одному домену:

- один UI-компонент
- один API-ендпоінт
- один баг в одному шарі
- один рефакторинг в одному модулі

Якщо задача вимагає кросдоменної координації (API-контракт + UI + QA), використовуйте [`Мульти-агентний проєкт`](./multi-agent-project.md).

## Передпольотний чек-лист

Перед промптом визначте:

1. точний результат (файл або поведінка)
2. цільовий стек та версії
3. критерії прийняття
4. очікування щодо тестування

## Шаблон промпту

```text
Build <specific artifact> using <stack>.
Constraints: <style/perf/security constraints>.
Acceptance criteria:
1) ...
2) ...
Add tests for: <critical cases>.
```

## Приклад промпту

```text
Create a login form component in React + TypeScript + Tailwind CSS.
Constraints: accessible labels, client-side validation, no external form library.
Acceptance criteria:
1) email and password validation messages
2) disabled submit while invalid
3) keyboard and screen-reader friendly
Add unit tests for valid/invalid submit paths.
```

## Очікуваний потік виконання

1. Відповідна навичка обирається автоматично.
2. Агент пропонує реалізацію та припущення.
3. Ви підтверджуєте або коригуєте припущення.
4. Агент створює код та тести.
5. Ви виконуєте локальну верифікацію та запитуєте невеликі доопрацювання.

## Контрольна точка якості перед злиттям

- поведінка відповідає критеріям прийняття
- тести покривають основний сценарій та ключові граничні випадки
- немає непов'язаних змін файлів
- немає прихованих зламних змін у спільних модулях

## Сигнали ескалації

Переходьте до мульти-агентного потоку, коли:

- робота з UI вимагає нових API-контрактів
- одне виправлення створює каскадні зміни між шарами
- обсяг виходить за межі одного домену після першої ітерації

## Критерії завершення

Виконання однієї навички завершено, коли:

- цільовий артефакт реалізовано
- критерії прийняття наочно виконані
- тести додані або оновлені для зміненої поведінки
