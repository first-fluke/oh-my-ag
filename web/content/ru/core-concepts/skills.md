# Навыки

## Прогрессивное раскрытие

Навыки выбираются на основе намерения запроса. Ручной выбор навыка обычно не требуется.

## Двухуровневая архитектура

Каждый навык использует **двухуровневую архитектуру с оптимизацией токенов**:

| Уровень | Содержимое | Размер |
|---------|-----------|--------|
| `SKILL.md` | Идентификация, условия маршрутизации, основные правила | ~40 строк (~800Б) |
| `resources/` | Протоколы выполнения, примеры, чек-листы, руководства, сниппеты, технический стек | Загружается по требованию |

Это обеспечивает **~75% экономии токенов** при начальной загрузке навыка (3-7КБ -> ~800Б на навык).

## Общий слой ресурсов (`_shared/`)

Общие ресурсы, дедуплицированные между всеми навыками:

| Ресурс | Назначение |
|--------|-----------|
| `reasoning-templates.md` | Структурированные шаблоны для многошагового рассуждения |
| `clarification-protocol.md` | Когда спрашивать, а когда предполагать; уровни неоднозначности |
| `context-budget.md` | Стратегии токен-эффективного чтения файлов по уровням моделей |
| `context-loading.md` | Маппинг типов задач к ресурсам для построения промпта оркестратора |
| `skill-routing.md` | Маппинг ключевых слов к навыкам и правила параллельного выполнения |
| `difficulty-guide.md` | Оценка сложности (Простая/Средняя/Сложная) с ветвлением протоколов |
| `lessons-learned.md` | Накопленные между сессиями доменные особенности |
| `verify.sh` | Автоматический скрипт верификации после завершения работы агента |
| `api-contracts/` | PM создаёт контракты, backend реализует, frontend/mobile потребляют |
| `serena-memory-protocol.md` | Протокол чтения/записи памяти в режиме CLI |
| `common-checklist.md` | Универсальные проверки качества кода |

## Ресурсы отдельных навыков

Каждый навык предоставляет доменно-специфичные ресурсы:

| Ресурс | Назначение |
|--------|-----------|
| `execution-protocol.md` | 4-шаговый процесс цепочки рассуждений (Анализ -> План -> Реализация -> Верификация) |
| `examples.md` | 2-3 примера формата ввод/вывод |
| `checklist.md` | Доменно-специфичный чек-лист самопроверки |
| `error-playbook.md` | Восстановление после ошибок с правилом эскалации «3 попытки» |
| `tech-stack.md` | Подробные спецификации технологий |
| `snippets.md` | Готовые к использованию шаблоны кода |

## Почему это важно

Это позволяет сохранять начальный контекст компактным, обеспечивая при этом глубокое выполнение при необходимости.
