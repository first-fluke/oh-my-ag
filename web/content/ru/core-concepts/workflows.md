# Рабочие процессы

## Команды рабочих процессов

- `/coordinate`
- `/orchestrate`
- `/plan`
- `/review`
- `/debug`

## Навыки и рабочие процессы

- Навыки: автоматически активируются на основе намерения запроса
- Рабочие процессы: явные многошаговые конвейеры, запускаемые пользователем

## Типичная последовательность мультиагентной работы

1. `/plan` для декомпозиции
2. `/coordinate` для поэтапного выполнения
3. `agent:spawn` для параллельных субагентов
4. `/review` для контрольной проверки QA
