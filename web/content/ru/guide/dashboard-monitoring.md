# Сценарий: Мониторинг через панель управления

## Команды запуска

```bash
bunx oh-my-ag dashboard
bunx oh-my-ag dashboard:web
```

URL веб-панели по умолчанию: `http://localhost:9847`

## Рекомендуемая компоновка терминалов

Используйте минимум 3 терминала:

1. терминальная панель мониторинга (`oh-my-ag dashboard`)
2. команды запуска агентов
3. логи тестов/сборки

Держите веб-панель открытой для общей видимости во время командных сессий.

## Что отслеживают панели

Источник данных: `.serena/memories/`

Основные сигналы:

- статус сессии (`running`, `completed`, `failed`)
- назначения на доске задач и изменения состояний
- ходы прогресса для каждого агента
- события публикации результатов

Обновления управляются событиями при изменении файлов; полный цикл опроса директории не требуется.

## Руководство по реагированию: сигнал -> действие

- `No agents detected`
  - убедитесь, что агенты запущены с тем же `session-id`
  - проверьте, что `.serena/memories/` записывается
- `Session stuck in running`
  - проверьте временные метки последних файлов `progress-*`
  - перезапустите заблокированного агента с более чётким промптом
- `Frequent reconnects (web)`
  - проверьте локальный файрвол/прокси
  - перезапустите `dashboard:web` и откройте страницу заново
- `Missing activity while agents are active`
  - убедитесь, что записи оркестратора не перенаправлены в другое рабочее пространство

## Чек-лист мониторинга перед слиянием

- все необходимые агенты достигли состояния completed
- нет неразрешённых критических замечаний QA
- файлы результатов присутствуют для каждого агента
- интеграционные тесты выполнены после финальных выходных данных агентов

## Критерии завершения

Фаза мониторинга завершена, когда:

- сессия достигла терминального состояния (`completed` или намеренно остановлена)
- история активности объясняет происхождение финального результата
- решение о релизе/слиянии принято с полной видимостью статуса
